## 要点

- 期限结构模板利用 Endless 数据不同时期预期值差值，捕捉分析师预估的公司成长性，可通过不同时间期限指标相减实现。
- 单因子模型用市场收益率预估股票收益率，后人加入风格因子构建资产组合优化理论。
- 主平均差计算每个股票数据字段指标与所在组平均值的差值，体现股票相对组的差异。
- 时间导数的 idea 可将数据字段类比物理概念，通过 TS 德尔塔算子计算变化量、速度和加速度。
- 分组排名差通过子行业排名与全市场排名相减，修正公司在市场上的排名。
- 两个指标排名差可找出两个数据字段在时间上变化不一致的情况，提供有价值信号。
- 检查因子自相关性可降低 prod correlation 检验的因子数量，提升效率，避免平台限流。
- 计算因子相关性的工作流程包括获取阿尔法 PNL 和 ID、准备数据、计算相关性等步骤。
- 推送回测需注意版本和卡槽使用，可通过设置全局变量、增加 sleep 机制等解决报错问题。
- 阿尔法 PNL 可视化可筛选表现最好的因子，避免欠拟合，因子稳健性影响 OS 表现和 base payment。
- Genius 项目从阿尔法数量、分散性、简单性、稳健性和社区贡献五个维度指引因子挖掘方向。
- 因子稳健性影响因子在 OS 上的表现和 base payment，有工具可进行稳健性分析以避免提交 OS 表现差的阿尔法。
- Genius 项目从 Alpha number、分散性、Simpelicity、稳健性、社区活动五个维度为顾问提供因子挖掘方向指引。
- Genius 浏览器插件支持所有 chromium 内核浏览器，获取方式有 Git clone 到本地、手动下载、解压群里的 ZIP 压缩包。
- 安装插件需解压文件，进入浏览器管理扩展页面，打开开发者模式，点击加载解压缩的扩展，选择解压文件夹内的子文件夹。
- 插件功能包括运算服务分析、排名分析和批量点赞，运算服务分析可检查运算符使用情况，排名分析可发现自身薄弱项并预估级别。
- 批量点赞功能可在论坛为中国区顾问点赞，需启用隐藏功能，保存设置后进入顾问专属中文论坛操作。
- 阿尔法是 data 与 operator 的组合，好的阿尔法应稳定，其生命周期包括样本内（IS）和样本外（OS），OS 又分半样本外（SEMIOS）和真实样本外。
- 过度拟合（overfitting）的阿尔法 OS 表现大概率差，稳健的阿尔法应如绿线表现，蓝线是信号走平的常见情况。
- 市场上阿尔法存在二八定律，80% 亏钱，20% 赚钱，提交的阿尔法保持赚钱趋势可使 value factor 不低。
- 提升阿尔法质量常见策略有重质量（提交三个好阿尔法）和重数量（提交 100 个阿尔法）。
- 每天拿到的钱反映 OS 情况，若 value factor 为 0.5，交一个阿尔法拿到 1.5 美元以下，OS 大概率亏钱。
- 做出阿尔法后需适当调整参数和添加运算符，但不能过度折磨数据。
- 练习修改阿尔法模板时，要注意 bucket 分组范围、参数短期概念、交易日概念、运算符作用及数据字段默认值等问题。
- base payment 由 quantity factor、quality factor、value factor、self growth factor 四个因素乘以权重组成，value factor 决定收入上限。
- 季度奖金由 Genius program 保证保底收入，上限由 value factor 和 which factor 综合决定，初期 value factor 影响大。
- 平台核心规则是多样性、高质量、高数量，成为顾问后要关注阿尔法质量。

## QA

### Q: 什么是期限结构模板？
A: 期限结构模板利用 Endless 数据不同时期预期值的差值，术语为期限结构（term structure）。它能捕捉不同时间期限内估计值的成长幅度，通过同一指标不同时间期限相减实现，体现分析师预估的公司成长性。

### Q: 单因子模型的原理是什么？
A: 单因子模型基于股票收益率由市场收益率和自身收益率构成的假设，用市场收益率预估股票收益率。公式简化后为贝塔乘上市场收益率，贝塔是股票与市场的相关系数。

### Q: 主平均差有什么作用？
A: 主平均差计算每个股票数据字段指标与所在组平均值的差值，类似用个人成绩减去班级平均分体现个人成绩价值，可体现股票相对于所在组的差异。

### Q: 如何理解时间导数的 idea？
A: 可将数据字段类比物理概念，原始值相当于位移，用 TS 德尔塔算子计算一段时间内的差值，反映变化量，除上常数可得速度概念，再加一个 TS 德尔塔可计算加速度。

### Q: 分组排名差是如何计算的？
A: 分组排名差是拿股票在子行业里的排名减去其在全市场上的排名，可对公司在市场上的排名进行修正，反映更准确的信息。

### Q: 两个指标排名差能提供什么信息？
A: 两个指标排名差可找出两个数据字段在时间上变化不一致的情况，即便单独表现不佳的两个表达式，相减后也可能表现出有价值的信号。

### Q: 检查因子自相关性有什么好处？
A: 检查因子自相关性可降低需要检验 prod correlation 的因子数量，减少对平台计算资源的请求频率，避免被限流，显著提升 check correlation 的效率。

### Q: 计算因子相关性的工作流程是怎样的？
A: 流程为：获取已提交的阿尔法的 PNL 和 Alpha ID 并保存为 PICO 文件；获取目标阿尔法全量信息；准备目标 Alpha 的 p PL 和日收益率，截取近四年数据；计算目标阿尔法与同 region 已提交阿尔法的皮尔逊相关性。

### Q: 推送回测时需要注意什么？
A: 普通版本回测需留出卡槽做稳健性测试，使用解耦模式需自己写推送机制。若出现报错，可检查 Python 及第三方包版本，设置全局变量、增加 sleep 机制等。

### Q: 如何进行阿尔法 PNL 的可视化？
A: 首先获取所有跑过的阿尔法的 PNL 数据，输出为 data frame。可根据需要选择查看或注释掉查看代码。设置绘图参数，绘制画布，根据因子表现筛选出表现最好的因子。

### Q: 因子稳健性对 OS 表现和 base payment 有什么影响？
A: 因子稳健性会影响其在 OS 上的表现，若因子在 IS 阶段参数不稳定，到 OS 阶段可能更不稳定。而 OS 表现会影响 base payment，因为大部分收益基于 OS 表现。

### Q: Genius 项目的作用是什么？
A: Genius 项目从阿尔法数量、分散性、简单性、稳健性和社区贡献五个维度给予顾问正确的因子挖掘方向指引，按其标准挖掘的因子往往更稳健，OS 表现更好。

### Q: 如何获取 Genius 浏览器插件？
A: 支持所有 chromium 内核的浏览器。获取方式有三种：Git clone 到本地；手动下载，进入指定 URL 下载；使用群里发送的 ZIP 压缩包。

### Q: 因子稳健性对因子表现和收入有什么影响？
A: 因子稳健性会影响因子在 OS 上的表现，进而影响 base payment。若因子在 IS 阶段参数不稳定，在 OS 阶段可能更不稳定，使用工具进行稳健性分析可避免提交 OS 表现差的阿尔法。

### Q: 如何获取和安装 Genius 浏览器插件？
A: 获取方式有三种：Git clone 到本地；手动进入 URL 下载；解压群里的 ZIP 压缩包。安装时，先解压文件，再进入浏览器管理扩展页面，打开开发者模式，点击加载解压缩的扩展，选择解压文件夹内的子文件夹。

### Q: Genius 浏览器插件有哪些功能？
A: 插件功能包括运算服务分析、排名分析和批量点赞。运算服务分析可检查运算符使用情况；排名分析可发现自身薄弱项并预估级别；批量点赞可在论坛为中国区顾问点赞。

### Q: 什么是好的阿尔法？
A: 好的阿尔法应是稳定的，其生命周期包括样本内（IS）和样本外（OS）。真实样本外是提交阿尔法之后的每一天，好的阿尔法应从提交后开始创造收益。

### Q: 过度拟合的阿尔法有什么后果？
A: 过度拟合（overfitting）的阿尔法 OS 表现大概率差，会出现类似红线的情况，收益为负。而稳健的阿尔法应如绿线表现。

### Q: 提升阿尔法质量有哪些策略？
A: 常见策略有重质量（提交三个经过仔细研究、有信心的阿尔法）和重数量（提交 100 个机器做出的阿尔法）。

### Q: 如何判断阿尔法的 OS 表现好坏？
A: 每天拿到的钱反映 OS 情况，若 value factor 为 0.5，交一个阿尔法拿到 1.5 美元以下，OS 大概率亏钱，具体数值可在论坛搜索。

### Q: 做出阿尔法后如何调整？
A: 做出阿尔法后需适当调整，从 idea 角度和参数方面调整，但不能过度折磨数据。可先用测试稳定的工具，积累经验后再思考添加运算符提升 performance。

### Q: 修改阿尔法模板时要注意什么？
A: 要注意 bucket 分组范围、参数短期概念、交易日概念、运算符作用及数据字段默认值等问题，避免出现常见错误。

### Q: base payment 由哪些因素组成？
A: base payment 由 quantity factor、quality factor、value factor、self growth factor 四个因素乘以权重组成，value factor 基本决定了收入上限。

### Q: 季度奖金是如何确定的？
A: 季度奖金由 Genius program 保证保底收入，上限由 value factor 和 which factor 综合决定。初期 value factor 影响大，随着 which factor 累积，其影响会变化。

### Q: 平台的核心规则是什么？
A: 平台核心规则是多样性、高质量、高数量，无重要性先后之分。成为顾问后要关注阿尔法质量，对自己负责。
